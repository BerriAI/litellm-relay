V0 product scope

Bring every employee AI tool back through LiteLLM Gateway.

LiteLLM Relay is a machine-installed agent that routes supported desktop apps, browser AI, coding tools, and MCP servers through the LiteLLM Gateway already trusted for governance, budgets, guardrails, and observability.

Review Notion AI path Open repo scope

The wedge is routing first, transparent capture second.

Do not promise perfect interception on day one. V0 wins by routing tools with base URL or proxy support, discovering shadow AI, and graduating high-value proprietary apps into managed capture.

Route

Supported tools use local endpoints

Expose OpenAI-compatible and Anthropic-compatible local endpoints, then forward through LiteLLM Gateway with device and team metadata.

  • Codex-style OpenAI clients
  • Claude Code and Anthropic-compatible CLIs
  • Aider, Continue, OpenCode, custom SDK apps
Discover

Shadow AI and MCP inventory

Scan app configs, MCP server files, process destinations, and proxy flows so admins can see what tools are active before enforcing policy.

  • Device app coverage
  • MCP server allowlist
  • Unsupported destination alerts
Capture

Managed proxy for proprietary apps

Use PAC/system proxy first. Add MITM and Network Extension only where customers accept the certificate, privacy, and deployment tradeoffs.

  • macOS PAC profile for V0
  • Root CA only for approved domains
  • Network Extension fallback

Notion AI: realistic proof path

The Notion Mac app is the right demo because it is a real desktop AI app. The honest V0 proof has two levels: capture metadata today, then decrypt and adapt only on a managed test Mac.

Local evidence from this machine

What we proved without installing anything

The Notion Mac app is installed and running as Electron. Its network service process has active HTTPS sockets. The current system has no configured HTTP proxy, so we did not change machine routing or inspect private content.

  • Process: Notion Helper --utility-sub-type=network.mojom.NetworkService
  • Observed flow: Notion PID 853 -> TCP :443
  • System proxy: disabled
  • App bundle exposes Notion AI shortcut handling and NetLog troubleshooting strings.

01

V0 demo: PAC/system proxy capture

Deploy Relay as 127.0.0.1:<port>, install a managed PAC profile for *.notion.so, *.notion.com, and api.notion.com, then trigger Notion AI. Success is Notion app traffic arriving at Relay with process, destination, timing, SNI or CONNECT host, and byte counts.

02

Safe shadow-route proof

Use a scratch Notion page and prompt marker such as PROBE-LITELLM-<uuid>. Relay redacts cookies, IDs, URLs, and raw prompt text, then sends a synthetic proof call to LiteLLM Gateway with the same hashed proof ID. This proves Gateway correlation without modifying Notion.

03

MITM adapter proof

On a managed test Mac, trust the Relay enterprise CA and capture only approved Notion domains. Look for Notion AI paths such as /api/v3/runInferenceTranscript. Convert the private request into a LiteLLM Gateway call, then synthesize a Notion-compatible response.

04

Production fallback

If the app bypasses system proxy or customers need process-level enforcement, move capture to macOS NETransparentProxyProvider. TLS decryption still requires a trusted CA and explicit customer policy.
